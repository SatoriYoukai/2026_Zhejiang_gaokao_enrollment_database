# 志愿落点概率估算小工具

这个工具用于估算浙江“专业平行志愿”表中每个志愿的到达概率和最终落点概率。

它适合做志愿表结构体检，例如：

- 前部彩票是否真正有检索机会；
- 哪个志愿会成为主要截流点；
- 稳段和保底是否足够厚；
- 调整志愿顺序后，落点概率如何变化。

它不适合把 `12.34%` 当成精确录取概率。三年历史数据太少，模型只能给出经验估计。

## 输入格式

支持三类输入：

- `.xls` 志愿录入模板格式；
- `.xlsx` 志愿录入模板格式；
- `.csv`，四列为 `院校代码/院校名称/专业代码/专业名称`，也兼容同义英文列名。

志愿录入模板的前四列应为：

```text
院校代码 | 院校名称 | 专业代码 | 专业名称
```

程序会用 `院校代码-专业代码` 去内置数据库匹配招生计划、历史位次和预测参考位次。

## 无历史位次口径

每条志愿的参考位次按以下顺序生成：

1. 有 2025 年历史位次：使用 2025 年最低录取位次。
2. 无 2025 年但有 2024 或 2023 年：使用最近一年历史位次。
3. 完全无历史：使用同校所有有历史专业的截尾均值。
4. 如果仍无法预测：运行时明确要求用户手动输入。

同校截尾均值规则：

- 同校有历史专业数大于 2 时，去掉最高位次和最低位次后取平均；
- 同校有历史专业数不大于 2 时，直接取平均。

运行时如果志愿表中出现完全无历史的预测位次，程序会先展示这些预测值并请求确认。用户可以接受，也可以选择全部手动输入。

## 运行方式

便携版用户双击：

```text
运行_志愿落点概率估算器.bat
```

macOS 便携版用户双击：

```text
运行_志愿落点概率估算器.command
```

命令行用户可以运行：

```powershell
志愿落点概率估算器.exe "你的志愿表.xls" --user-rank 39000
```

开发环境中运行：

```powershell
python -X utf8 tools\estimate_volunteer_landing_portable.py "outputs\volunteer_order_v1_20260629\volunteer_input_filled_20260630_latest_2.xls" --master data\rank_model_database.csv
```

常用参数：

- `--user-rank`: 考生位次，默认 `39000`。
- `--draws`: Monte Carlo 模拟次数，默认 `200000`。
- `--trend-mode`: 历史趋势处理方式。
- `--predicted-rank-policy`: 无历史预测位次处理方式，`ask` 为交互确认，`accept` 为全部接受，`manual` 为全部手填。
- `--progress`: 进度条模式，`auto` 为交互式终端显示，`on` 为强制显示，`off` 为关闭。
- `--progress-interval`: 进度条刷新间隔，默认 `0.5` 秒。

`trend-mode` 可选：

- `rankbin_centered`: 推荐。按位次段去掉 2023-2025 的整体趋势，只保留波动形状。
- `global_centered`: 去掉全局趋势。
- `raw`: 继承 2023-2025 的整体趋势。

## 输出

程序会在输入文件同目录生成：

- `*_落点概率估算.csv`
- `*_落点概率估算.xlsx`
- `*_落点概率估算报告.txt`

主要字段：

- `参考位次`: 用于估算的位次锚点。
- `参考位次来源`: 2025 历史、最近历史、同校截尾均值或用户手填。
- `到达概率`: 前面的志愿都未录取，因此能检索到这一条的概率。
- `到达后可录概率`: 如果已经检索到这一条，这一条能录取的条件概率。
- `最终落点概率`: 最终正好被这一条录取的概率。
- `累计录取概率`: 检索到这一条为止的累计录取概率。
- `该志愿后仍未录取概率`: 检索完这一条后仍未录取的概率。
- `模型备注`: 历史年数、小计划、参考位次来源和相似样本说明。

## 性能说明

便携版为了减小包体和降低跨平台依赖风险，使用标准库 Monte Carlo 内循环，没有依赖 NumPy。

在本机 80 条志愿表的粗略测试中：

- 读取输入、加载位次数据库、构建相似历史样本有几秒固定成本；
- 模拟部分主要是单核 CPU 计算，耗时大致随 `--draws` 线性增长；
- 默认 `200000` 次模拟通常是几十秒量级，而不是程序卡死。

进一步优化有两条路：

- 保持便携小包：优化样本索引和内循环，收益中等，兼容性风险低；
- 使用 NumPy/Pandas 向量化：速度会更快，但包体明显变大，macOS/Windows 打包和杀毒误报变量也更多。

## 数学建模简述

模型使用主数据库中 2023-2025 年的历史最低录取位次。

第一步，把位次转成全省参考人数下的百分位，再做 logit 变换：

```text
p = rank / population
z = log(p / (1 - p))
```

这样做是为了避免把不同位次段的同样绝对位次差看成完全等价。

第二步，构造历史年际残差：

```text
delta_z = z_next_year - z_prev_year
```

每个同院校专业最多贡献 `2023->2024` 和 `2024->2025` 两个样本。

第三步，对每个待估志愿，从历史残差中抽取相似样本。相似度主要考虑：

- 参考位次接近；
- 专业族相近，例如数学、统计、信计、计科、AI；
- 招生计划数相近；
- 浙江省内/省外。

第四步，对每个志愿生成可能的 2026 最低录取位次，并按专业平行志愿规则模拟：

```text
按志愿顺序检索；
第一个满足“考生位次 <= 模拟最低录取位次”的志愿即为本轮落点；
重复多次后统计概率。
```

对无历史、一年历史、小计划志愿，模型会额外增加不确定性。

## 解释注意

这个模型最有价值的输出不是单个概率的小数点，而是志愿表结构：

- 哪个志愿是主截流点；
- 后续志愿是否基本没有机会被检索；
- 稳保是否足够；
- 调整顺序是否显著改变落点分布。

它不能替代人工判断培养方案、学术平台、城市偏好和个人偏好。
